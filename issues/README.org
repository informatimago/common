* How this tracker works

Each issue is one file in =issues/open/= with the =.issue= extension and an
org-mode local-variables line, so it opens in org-mode in Emacs.  When an
issue is resolved, move the file to =issues/closed/= (e.g. =git mv=) and tick
its TODO.

Issues were produced by a code review of the whole tree on 2026-06-20
(GCC 14 / modern libobjc toolchain).  Severities: =critical= (memory-unsafe
or silently wrong results), =high=, =medium=, =low=.
