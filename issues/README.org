* How this tracker works

Each issue is one file in =issues/open/= (or =issues/closed/= once resolved)
with the =.issue= extension and an org-mode local-variables line.  When an
issue is resolved, move it to =issues/closed/= (=git mv=) with a RESOLUTION
note and tick its TODO -> DONE.

Issues were produced by a full-tree code review on 2026-06-20 (GCC 14 / modern
libobjc).  Severities: =critical= (memory-unsafe or silently wrong), =high=,
=medium=, =low=.
